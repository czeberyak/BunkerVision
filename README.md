# BunkerVision: Industrial AI Pipeline

Система автоматического мониторинга уровня заполненности бункеров сырьем (картофель) с использованием Computer Vision (YOLOv8n-cls) и паттернов отказоустойчивой архитектуры.

## Описание проекта
Проект разработан как Proof of Concept (PoC) для интеграции в производственную линию. 
Основной фокус сделан на **отказоустойчивость (Store-and-Forward)**, **работу в условиях экстремального дефицита данных** (синтез датасета) и **автономность работы** без постоянного контроля оператора.

## Ключевые архитектурные решения
1. **Region of Interest (ROI):** Программное отсечение соседних бункеров (cropping) для устранения шума на этапе препроцессинга RTSP-потока.
2. **Synthetic Data Generation:** Использование `albumentations` для имитации заводских условий (пыль, блики, RTSP-артефакты) — расширение датасета с 5 до 500+ кадров.
3. **Асинхронная интеграция (Store-and-Forward):** Локальный SQLite-буфер гарантирует сохранность телеметрии при падении локальной сети завода или недоступности ERP-системы.
4. **Автономный оркестратор:** Использование `APScheduler` для независимого поллинга камер и системы Telegram-алертов для мгновенного оповещения о критических уровнях (<5% или >95%).

## Структура репозитория
- `src/` — Ядро системы:
  - `capture.py` — Захват кадров (OpenCV) с обработкой таймаутов RTSP.
  - `model.py` — Обертка для инференса YOLOv8-cls.
  - `db.py` / `sender.py` — Локальный SQLite-буфер и модуль отправки HTTP POST в учетную систему.
  - `telegram.py` — Отправка тревожных уведомлений.
- `notebooks/` — R&D ноутбуки (выделение ROI, генерация синтетики, обучение модели).
- `monitor.py` — Главный оркестратор и точка входа для службы Windows.
- `cameras_config.json` — Конфигурация потоков, параметров ROI и доступов.

## Результаты PoC
- Модель: **YOLOv8n-cls** (1.4M параметров, инференс <5ms на CPU).
- Обучение: 10 эпох на синтетическом датасете.
- Точность (Top-1 Accuracy на hold-out выборке): **>99%**.

## Быстрый старт
Для развертывания системы в качестве службы Windows:
1. Установите зависимости: `pip install -r requirements.txt`
2. Настройте RTSP-ссылки и токены Telegram в `cameras_config.json`
3. Запустите `install_service.bat` для регистрации `monitor.py` через NSSM.
