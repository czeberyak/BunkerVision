# BunkerVision: Industrial AI Pipeline

Система автоматического мониторинга уровня заполненности бункеров сырьем (картофель) с использованием Computer Vision (YOLOv8n-cls) и паттернов отказоустойчивой архитектуры.

## Описание проекта
Проект разработан как Proof of Concept (PoC) для интеграции в производственную линию. 
Основной фокус сделан на **отказоустойчивость (Store-and-Forward)** и **работу в условиях экстремального дефицита данных** (использование статического ROI и пайплайна аугментаций для синтеза датасета).

## Ключевые архитектурные решения
1. **Region of Interest (ROI):** Программное отсечение соседних бункеров (cropping) для устранения шума на этапе препроцессинга.
2. **Pixel-based Baseline:** Использование HSV-масок для оценки физической площади сырья и калибровки нулевой линии без ML.
3. **Synthetic Data Generation:** Использование `albumentations` для имитации заводских условий (пыль, блики, RTSP-артефакты) — расширение датасета с 5 до 500 кадров.
4. **Асинхронная интеграция:** Локальный SQLite-буфер гарантирует сохранность данных (Telemetry) при падении сети или недоступности ERP-системы завода.

## Структура репозитория
- `src/` — Исходный код модулей (захват кадров OpenCV, инференс, БД, отправка HTTP POST).
- `notebooks/` — R&D ноутбуки (выделение ROI, цветовой baseline, генерация синтетики, обучение модели).
- `monitor.py` — Главный оркестратор (точка входа для Windows Service).
- `cameras_config.json` — Конфигурация RTSP-потоков и координат ROI для камер.

## Результаты PoC
- Модель: **YOLOv8n-cls** (1.4M параметров, inference <5ms на CPU).
- Обучение: 10 эпох на синтетическом датасете.
- Точность (Top-1 Accuracy на hold-out выборке): **>99%**.

## Быстрый старт
Для развертывания системы в качестве службы Windows:
1. Установите зависимости: `pip install -r requirements.txt`
2. Настройте RTSP-ссылки в `cameras_config.json`
3. Запустите `install_service.bat` для регистрации `monitor.py` через NSSM.
